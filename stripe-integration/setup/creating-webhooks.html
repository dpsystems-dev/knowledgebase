How to Create a Stripe Webhook

Webhooks allow Stripe to notify your portal when payment events occur (e.g., payment completed, payment failed). This guide walks you through creating a webhook endpoint.

How to Access the Webhooks Section​

What you'll do: Navigate to the webhook configuration in Stripe.

Step 1: Open Developers Menu​

  1. In the Stripe Dashboard, click "Developers" in the bottom-left corner of the screen

  2. A menu will expand. Click "Webhooks" from the list

Developers menu expanded showing Webhooks option

How to Add a New Webhook Destination​

What you'll do: Start creating a new webhook endpoint.

Step 2: Add a New Destination​

  1. On the Event destinations page, click the green "+ Add destination" button in the top-right area

Webhooks page showing the Add destination button

How to Select Webhook Events​

What you'll do: Choose which payment events should trigger notifications.

Step 3: Select Events to Listen For​

  1. Under "Events from", select "Your account" (the first option with the diagram icon)

  2. In the Events section, you'll see a search box. Type checkout in the search box

  3. Check the following 2 required events:

    • checkout.session.completed
    • checkout.session.expired

    Optional: You can also add payment_intent.payment_failed for explicit failure notifications, though card declines are typically handled through the checkout session flow.

  4. Click the "Continue" button

Event selection showing checkout events checked

How to Choose the Destination Type​

What you'll do: Select webhook endpoint as the destination.

Step 4: Choose Destination Type​

  1. Select "Webhook endpoint" (the first option - "Send webhook events to a hosted endpoint")

  2. Click "Continue"

Destination type options with Webhook endpoint selected

How to Enter Your Endpoint URL​

What you'll do: Enter the URL where Stripe should send webhook events.

Step 5: Enter Your Endpoint URL​

  1. In the "Endpoint URL" field, enter your portal's webhook URL

Format: {your-baseUrl}/api/payments/webhook

Example: If your portal is at https://web-dp.tenjo.ovh, enter:

https://web-dp.tenjo.ovh/api/payments/webhook

  1. The "Destination name" field is auto-generated - you can leave it as-is or change it

  2. The "Description" field is optional

  3. Click "Create destination"

Endpoint URL field with example URL entered

important

The URL must:

  • Start with https://
  • Be accessible from the internet (not localhost)
  • End with /api/payments/webhook

How to Get the Webhook Secret​

What you'll do: Copy the webhook signing secret for portal configuration.

Step 6: Get the Webhook Secret​

After creating the webhook, you'll be taken to the destination details page.

  1. Find the "Signing secret" section on the right side of the page

  2. Click the eye icon to reveal the secret

Signing secret section with eye icon to reveal

  1. Click the clipboard icon to copy the secret to your clipboard

Signing secret revealed with copy button

  1. The secret should start with whsec_. Save this value - you'll need it for the portal configuration
tip

The webhook secret is used to verify that incoming webhook requests actually came from Stripe and weren't forged by a malicious third party.

Required Webhook Events Summary​

EventPurpose
checkout.session.completedNotifies when a payment succeeds
checkout.session.expiredNotifies when a checkout session times out
payment_intent.payment_failed(Optional) Explicit card decline notifications

Next Steps​

After copying your webhook secret:

  1. Configure the Portal - Enter the webhook secret in Admin Dashboard
